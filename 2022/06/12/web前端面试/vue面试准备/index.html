vue面试准备

2022-06-12 vue, 前端面试 Comments

趋势

  • Vue 与 React 越来越接近
    • Vue3 Options API 对应 React class Component
    • Vue3 Composition API 对应 React Hooks

Vue 使用

概述
  • 基本使用，组件使用——常用，必会
  • 高级特性——不常用，体现深度
  • Vuex 和 Vue-router使用
Vue 基本使用

  • 指令、插值

    • 插值、表达式

    • 指令、动态属性：:id

    • v-html：会有XSS风险，会覆盖子组件（好像新版本的vue加了校验，没这个问题了）

  • computed 和 watch

    • computed 有缓存，data 不变则不会重新计算
    • watch 如何深度监听？
    • watch 监听引用类型，拿不到 oldVal

  • class 与 style

  • v-if 和 v-show 条件渲染

    • v-if 是真正的条件渲染（也就是改变属性会重新渲染），初始渲染较快（少渲染DOM了），后续改变属性开销大
    • v-show 只是改变了 display 属性，也就是第一次加载把DOM全渲染了，后续只改变display:none罢了

  • v-for 循环（列表）渲染

    • key 不能乱写
      • 尽量不用 index 做 key（好像没问题？）
    • v-for 和 v-if 不能一起使用
      • 应该把 v-if 加在 v-for 的外层，也不知道为啥有人能把这俩一起用

  • 事件

    • event 参数，自定义参数
      • event 是原生的事件
      • 事件挂载到当前元素
    • 事件修饰符，按键修饰符
      • 事件修饰符：stop 阻止单击事件继续传播； prevent 提交事件不再重载页面； capture 使用捕获模式； self 只处理元素自发触发的事件
      • 按键修饰符：ctrl.exact 只有ctrl 按下时事件才能被触发

  • v-model 表单

    • 修饰符：lazy 输入完成后改变data； trim 去除首尾空格； number 自动把输入解析成number给到data（解析好像不咋地，很难用）
    • 输入框、复选框、多个复选框、单选框、下拉列表
Vue组件使用
  • props 和 $emit （父子组件通信）
    • props 用来接收父组件传递的data，（父组件使用动态属性传递data）
      • props 可以搭配 computed 来对单向流动的数据做出更改（比如 trim 去空格，toLowerCase 转化小写）
    • $emit 用来触发定义好的父组件上的函数
  • 组件间通讯 - 自定义事件 （兄弟组件通信）
    • Vue3不支持这么做，因为不利于维护，可以选择子组件通过父组件建立联系
  • 组件生命周期
    • 创建，改变，销毁
    • created mounted beforeUpdate updated unmounted
Vue高级特性

  • 自定义 v-model

    • 就是自定义的组件使用 v-model 来像 form 组件一样方便获取数据
    • 感觉是个语法糖，和父子组件传参差不多

  • $nextTick

    • Vue 是异步渲染，data改变后，DOM不会立刻渲染
    • 因此有些同步函数想在渲染执行后生效（渲染是异步）就需要被包裹在 $nextTick 里（完全没用过）

  • slot

    • 基本使用
    • 作用域插槽
    • 具名插槽

  • 动态、异步组件

    • 动态组件

    • 异步组件：加载组件时再去请求资源。有些组件并不是一定要加载的，只在必要时加载（v-if）能提高网络效率

  • keep-alive 缓存组件

  • mixin

    • 多个组件有相同的逻辑，抽离出来
    • Vue 3 提出了 Composition API 来解决这些问题
    • 存在的问题
      • 变量来源不明确，不利于阅读（需要点开对应的mixin文件查看）
      • 多 mixin 可能会造成命名冲突（同名data之间相互干扰）
      • minxin 和组件可能会出现多对多的关系（应该从降低复杂度的角度出发）

  • 组件 data 的数据一旦变化，立刻触发视图的更新

虚拟DOM 和 diff 算法
  • 本质上是为了减少 DOM 操作。个人理解：这个东西是工程化的必然发展，把开发者的操作更多转移到了逻辑层，开发者可以不再关注DOM操作，有助于构建复杂逻辑层
  • vdom
    • snabbdom （某种vdom的实现方式）
      • h函数：用 JS 模拟 DOM 树（生成vdom树结构的算法）
      • vnode 数据结构（JS对象 模拟 DOM结构）
      • patch 函数 （比较两个 vdom 树差异的函数）
    • 虚拟DOM（vdom），用JS模拟一棵DOM树，vdom树更新的时候自动比对与原vdom树的差别，自动计算出需要更新的DOM，只做少量改变
  • diff 算法 （ patch 函数中的部分）
    • 是什么：快速对比两棵 vdom 树差异的算法。属于是核心难点
    • 如何把两个 vdom 树 的校验的时间复杂度降为O(n)
      • 只比较同一层级，不跨级比较
      • tag 不同，则直接删掉重建，不做深度比较（符合DOM操作的习惯）
      • tag 和 key 都相同，则认为当前节点是同一节点，不再进行深度比较。所以设置 key 是为了提高比较效率
模板编译（没看完）

  • 前置知识：JS 的 with 语法（打破作用域规则，易读性变差）

    • 如果 with 语法中访问 obj 中没有属性，就会直接报错，而不是返回 undefined

  • vue template complier 将模板编译为 render 函数

  • 执行 render 函数生成 vnode

Vue3 实现响应式

    • 读取嵌套的 obj 的对应的 key 时候才会变为响应式，而不是Object.defineProperty 直接把 obj 内部在初始化的时候就变成响应式
  • Reflect 作用
    • Reflect不是一个构造函数，应当作为对 obj 操作的工具函数
