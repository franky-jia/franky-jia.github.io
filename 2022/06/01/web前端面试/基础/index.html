HTTP协议
  • 主要特点
    • 无连接（没有持续化连接）
    • 无状态（无法确定身份）
  • HTTP报文的组成部分
    • 请求报文
      • 请求行（header里的第一行）、请求头（header除了第一行）、空行、请求体
    • 响应报文
      • 状态行（header里的第一行，有状态码）、响应头、空行、响应体
  • POST 和 GET的区别
    • GET在浏览器回退时是无害的，而POST会再次提交请求
    • GET产生的URL地址可以被 收藏 或者被浏览器缓存，POST不可以
    • GET请求只能进行 ur l编码，而POST支持多种编码方式
    • GET有长度限制，并且只能是ASCII码，POST没有限制
    • GET参数通过URL传递，POST放在Request body中
  • HTTP状态码
    • 1XX：指示信息—表示继续处理
    • 2XX：成功
    • 3XX：重定向
    • 4XX：客户端错误
    • 5XX：服务器错误
    • 200：客户端请求成功
    • 206：客户发送了一个带有Range头的GET请求，服务器完成了它
    • 301：请求页面临时转移至新的 url
    • 304：客户端存在缓存文档，服务器告诉客户端可以使用缓存文件
    • 403：资源被禁止访问
    • 404：请求资源不存在
    • 500：服务器不可预知错误
    • 503：服务器过载
  • 持久连接
    • 使用 Keep-Alive 模式
  • 管线化
    • 用处不大，在持久连接的情况下，可以通过把分批次的请求和返回打包成一个批次的请求和返回，只有GET与 HEAD才可以

  • 类与继承

    1. 借助构造函数实现继承，子级构造函数直接引入父级构造函数并使其this改变指向。缺点是在父级构造函数的原型链上添加方法属性无法被子级构造函数继承，很显然

    2. 原型链继承，子类可以继承父类原型链上的方法，因为把父类的实例挂载到了子类的原型链（__proto__）上了。缺点是所有子类都会受到原型链变动的影响，当子类改变原型链上的对象或是数组里的内容时候，其他子类也会收到影响（需要注意在该属性不存于数组或者对象时候，子类会自己生成该属性或者方法）

    3. 组合继承方式（通常方式），结合上面的两种方法。在原型链继承的基础上，把带obj的父类通过构造函数的继承方式（直接继承）来继承

    4. 组合继承的优化1。new实例的方式会使得 Child的实例 的 proto 指向的是 Parent 的实例，造成的问题是，parent中的属性方法会在 Child这个层级存在，在Parent这个层级也存在，因此Parent这个层级的属性和方法是多余的，直接把Child挂载到Parent的原型链上即可

  1. 组合继承的优化2（完美方法）。在优化1的基础上实现了 子类 在 父类 原型链的下一级（父类此时由 Object.create()创建，是个空的实例）。我发现原型这个说法是不太好的，感觉叫做上一级实例会更好一些。最后再修改 constructor 就能足够完善了，也因为多了一个层级，所以可以自由修改 constructor

原型链

  • 创建对象有几种方法

  • 原型、构造函数、实例、原型链

    • 原型对象
      • 原型的 constructor 指向构造函数，构造函数的 prototype 指向 原型
    • 构造函数
      • 能用来 new 的就是构造函数，（也可以是不通过new的语法糖形式）
    • 实例
      • new 出来的就是实例。实例的 __proto__ 指向的原型
    • 原型链
      • 通过 __proto__ 来实现向上查找
      • 构造函数的prototype指向原型对象，所以new出来的实例的 __proto__ 指向原型对象

  • instanceof 的原理

    • 构造函数的 prototype 与 实例的 __proto__ 指向同一个

  • new运算符，

    • 记录一下 call apply bind 的区别

      • 功能都是把函数绑定到某个object上，同时函数中的this指向当前的object。call 一个参数是object，后面参数是函数所需参数，会立即执行。apply 第一个参数是object，后面参数是函数所需参数用[ ]包裹，会立即执行。bind 第一个参数是object，后面参数是函数所需参数，不会立即执行，而是返回一个this指向object的函数。好像没啥玩意，只要会一个 bind 就足够了，会call 可能会省代码

    • 手写 new：

      • Object.create(fn.prototype) 创建一个指向 fn.prototype（fn的原型） 的 空的实例
      • fn.apply(o, rest) 是把 fn 绑定到原型链上，o已经在原型链上了，让fn里的this指向o就让通过fn生成的实例挂载到了fn原型的下面，也就是新的实例的 __proto__指向 fn的原型
