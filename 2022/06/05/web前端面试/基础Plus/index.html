异步进阶

  • event loop，比较关键的点在于，异步指令不是立即放入异步队列的，其他人很少能讲到。感觉可以把 Call Stack 理解为同步队列，Callback Queen 理解为异步队列

    • 同步代码，一行行放在 Call Stack 里面
    • 遇到异步，先“记录”，等待时机（比如定时器结束、网络请求收到回复），时机到了就移动到 Callback Queen
    • 当 Call Stack 空了，event loop 就从 Callback Queen 取出第一条指令放到Call Stack 开始执行。重复这个过程

  • DOM事件基于 event loop 但不是异步（也许是因为没有等待的过程，DOM事件触发了就直接把事件放入 Callback Queen）

  • Promise

    • 三种状态
      • pending fulfilled rejected
    • 状态的表现与变化
      • pending -> resolved
      • pending -> rejected
      • 变化不可逆
      • pending 状态，不会触发 then 和 catch
      • resolved 状态，会触发后续的 then
      • rejected 状态，会触发后续的 catch
    • then 和 catch 对状态的影响
      • then 正常返回 resolved，里面有报错返回 rejected
      • catch 正常返回 resolved，里面有报错返回 rejected
  • async/await
    • 同步语法，彻底消灭回调函数
    • async/await 和 Promise 的关系
      • 执行 async 函数，返回的是 Promise 对象
      • await 相当于 Promise 的 then，可以取到Promise中的的data，同时会把 await 后面的非 Promise 转化为 Promise
      • try…catch 可捕获异常，代替了 Promise 中的 catch

  • 异步的本质

    • async/await 实际上是种语法糖，写法像是同步，本质还是异步。不知道有什么用，这东西感觉不利于代码可读性
    • await 后面的所有代码块均看作异步，await 那行代码立即执行

  • for of （for in）异步的循环，区别于forEach 同步的循环

    • for of 与 for in 前者遍历元素，后者遍历下标，都是异步的循环
    • forEach 是同步的循环

  • 宏任务（macroTask） 和 微任务（microTask）

    • 宏任务：setTimeout，setInterval，Ajax，DOM事件（append渲染）
    • 微任务：Promise async/await
    • 微任务执行时机比宏任务要早
    • event-loop 和 DOM 渲染的关系

  • 微任务 和 宏任务的根本区别

    • 微任务是 ES6 语法规定的
    • 宏任务是由浏览器规定的

  • 手写 Promise 暂时忽略

HTTP

  • method

    • get 获取数据
    • post 新建数据
    • patch/put 更新数据
    • delete 删除数据

  • Restfu API

    • 传统API设计：把url当作一个功能（/api/list?pageIndex=2）
    • Restful API设计：把url当作一个唯一资源（/api/list/2）
      • 尽量不使用url参数
      • 用 method 表示操作类型

  • http headers

    • Host
    • cookie
    • Connection: keep-alive 一次TCP连接重复使用
    • User-Agent 浏览器信息
    • Content-type 数据格式
    • 自定义header

  • http缓存

    • 意义：网络效率是页面加载速度最重要的事

    • 静态资源可以缓存（js、css、img）

    • cache-control 强制缓存（资源多长时间内有效）

      • max-age
      • no-cache

    • 协商缓存（需要访问服务器）

      • last-modified 资源最后修改时间，每次网络请求会带上这个，如果资源是最新，服务器返回304
      • Etag 资源标识，用来标识资源，查看是否变动

    • 页面刷新对 http 缓存的影响

      • 三种刷新操作
        • 正常操作：地址栏输入url，跳转链接、前进后退
          • 强制缓存有效，协商缓存有效（正常使用缓存）
        • 手动刷新：F5，刷新按钮
          • 强制缓存失效，协商缓存有效（刷新不使用强制缓存，或者说不使用本地缓存）
        • 强制刷新：ctrl + F5
          • 强制缓存失效，协商缓存失效（强制刷新不使用缓存）

  • https

    • http是明文传输，https是密文传输
    • 加密方式：
      • 对称加密：同一个Key加密解密就是对称加密（成本低）
      • 非对称加密：公钥私钥是非对称
    • https证书
      • 使用第三方证书可以防范中间人攻击（不能防止窃听，不能防止线路干扰，可以防止中间人传递错误信息）
