Django学习

2022-05-16 Django Comments
环境安装：

  • virtualenv 有点像前端的 webpack

    • 作用

      • 使应用开发环境独立，与全局环境的python独立
      • 防止包管理混乱与版本冲突

    • 各种命令行操作：

      • pip install virtualenv 安装 virualenv `
      • virtualenv test 创建虚拟环境test
      • cd test 进入工程，里面的内容：
        • activate.bat 运行虚拟环境
        • deactivate.bat 退出虚拟环境
        • 虚拟环境中用 pip list 可以看到虚拟环境中的安装的python包

    • 安装 virtualenvwrapper 一种管理 virtualenv 虚拟环境的工具(安装之后就不需要进目录才能启动虚拟项目了)

      • pip install virtualenvwrapper-win 注意下载 windows 版 本的
      • mkvirtualenv test2 创建虚拟环境test2（自动在C盘的当前用户的目录下创建），创建完成后会自动进入虚拟环境
      • workon 查看所有由 virtualenvwrapper 管理的虚拟环境
      • workon test 运行虚拟环境 test

  • pycharm 开发工具，安装专业版，菜单栏 run->edit可以更改本地服务配置

  • navicat 数据库可视化管理工具

    • 启动数据库服务
      • net start mysql 数据库名字未必是mysql，安装的时候的自命名
      • net stop mysql 关闭数据库服务
      • net start 查看已启动服务
django 基础知识

  • 目录结构

    • settings.py 全局配置
    • url.py 总的路由，用来指向app应用
    • wsgi.py django的 wsgi接口，开发不用管
    • manage.py 包含 django 的所有命令
    • 创建app:
      • 菜单栏选择 tool->run massage.py task 在控制台输入startapp app01 创建app01
    • 创建其他文件夹
      • static 存放 网站所有的静态文件
      • log 存放 日志文件
      • media 存放 用户上传文件

    一个小常识，根搜索路径，在settings.py里面，设置根搜索路径可以方便写import，跟webpack用法好像一样
