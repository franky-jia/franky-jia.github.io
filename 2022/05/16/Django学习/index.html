Django学习

2022-05-16 Django Comments
环境安装：

  • virtualenv 有点像前端的 webpack

    • 作用

      • 使应用开发环境独立，与全局环境的python独立
      • 防止包管理混乱与版本冲突

    • 各种命令行操作：

      • pip install virtualenv 安装 virualenv `
      • virtualenv test 创建虚拟环境test
      • cd test 进入工程，里面的内容：
        • activate.bat 运行虚拟环境
        • deactivate.bat 退出虚拟环境
        • 虚拟环境中用 pip list 可以看到虚拟环境中的安装的python包

    • 安装 virtualenvwrapper 一种管理 virtualenv 虚拟环境的工具(安装之后就不需要进目录才能启动虚拟项目了)

      • pip install virtualenvwrapper-win 注意下载 windows 版 本的
      • mkvirtualenv test2 创建虚拟环境test2（自动在C盘的当前用户的目录下创建），创建完成后会自动进入虚拟环境
      • workon 查看所有由 virtualenvwrapper 管理的虚拟环境
      • workon test 运行虚拟环境 test

  • pycharm 开发工具，安装专业版，菜单栏 run->edit可以更改本地服务配置

    • pycharm也有数据库可视化的功能，好像可以代替navicat。pycharm 确实好用，虽然臃肿但是功能完善好用

  • navicat 数据库可视化管理工具

    • 启动数据库服务
      • net start mysql 数据库名字未必是mysql，安装的时候的自命名
      • net stop mysql 关闭数据库服务
      • net start 查看已启动服务
django 基础知识

  • 目录结构

    • settings.py 全局配置
    • url.py 总的路由，用来指向app应用
    • wsgi.py django的 wsgi接口，开发不用管
    • manage.py 包含 django 的所有命令
    • 创建app:
      • 菜单栏选择 tool->run massage.py task 在控制台输入startapp app01 创建app01
    • 创建其他文件夹
      • static 存放 网站所有的静态文件
      • log 存放 日志文件
      • media 存放 用户上传文件

    一个小常识，根搜索路径，在settings.py里面，设置根搜索路径可以方便写import，跟webpack用法好像一样

  • 快速搭建

    • 连接数据库的配置在settings里面，文档：https://docs.djangoproject.com/en/4.0/ref/settings/#databases
    • 然后是执行连接，tool->run massage.py task的控制台里面分别执行 makemigrations 和 migrate 会在连接数据库后生成默认表
      • 期间遇到个问题，安装了mysqlclient来解决（在此项目的虚拟环境中安装）。这个和范例的安装方式（范例安装的是mysql-python）不同，但是也成功连接上了数据库，有个问题在于pycharm中没有生成数据库的图标。
      • 更正，mysqlclient 是当前合适的解决方案，官方文档推荐
    • urls.py 配置总路由
    • views.py 设置view层，就是渲染页面的代码
    • settings.py 此时可以配置通用的查找资源路径（url中提及到了关键字之后会去这个设定好的路径对应的文件中去找资源，算是一个通用资源的配置访问方法）

  • models.py

    • 管理数据库用的，django实现了用操作 class 的方式来操作数据库
    • 先去写 自己的 app 中 model 中的class 。文档：https://docs.djangoproject.com/en/4.0/topics/db/models/#module-django.db.models
      • 常见的类型：
        • CharField
        • DateTimeField
        • EmailField
        • IntegerField
        • IPAddressField
        • FileField
        • ImageField
      • 常见的参数：
        • verbose_name
        • default=''
        • null=True
        • max_length
        • black=True
        • primary_key=True
      • 可以在内部添加Meta ，用来定义数据表的信息
    • 然后把自己的 app 加进 settings 中的 INSTALLED_APPS里面
    • 然后在 manage 窗口中 依次执行 makemigrations 'appname' 和migrate，就在对应的数据库中生成了新的对应自定义class的表单
    • 在数据表中添加字段
      • 先是实例化models.py中对应数据表的 class 直接对这个 class 进行数据更改，再 model.save() 就行，数据库的连接自动在 settings.py 里面做了
      • 字段存在中文需要添加 # _*_ encoding:utf-8 _*_
    • 在页面显示数据库中字段
      • 通过render 函数在渲染html页面的时候传递参数，在html 中有特定写法用来显示传递的参数（可以类似于jsx这种），很显然这个html文件是要通过特定的渲染方式最后才能生成真正的html 页面。我这里有个问题，如果和 jsx 类似的话，那直接引入 jsx 的渲染方式来在 django 里写 jsx 好像也可以。这样做好像有点南辕北辙，毕竟框架的存在是为了方便写代码，集成的渲染方式是框架的一部分
      • HTML 中的内置函数
        • 用于很方便生成 html 代码，包括循环什么的，和jsx很像
        • https://docs.djangoproject.com/en/4.0/ref/templates/builtins/
      • 像这种方式应该是前后没有分离的开发方式，也未必要学得多深入吧，后面继续学习 restful 接口看看分离是怎么做的

git status 查看目前的状态，以及可以执行的指令

有状态请求和无状态请求：

  • 无状态请求是不带多余参数的请求
  • 有状态请求是携带 cookie 的请求方式
    • cookie 是一种浏览器机制，浏览器会记录一些当前域名服务器分配的键值对，在访问对应域名时携带，用于记录用户的特征（让服务器知道是哪个用户发送的请求，可以用于记录用户登陆状态，个人设置，用户的数据等）
      • session 是对 cookie 的一种扩充形式，把用户的特征存储到了服务器上。用 cookie 来判断当前访问的用户，再去 session 上找对应的个人信息
