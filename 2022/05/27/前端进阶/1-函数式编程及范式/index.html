1-函数式编程及范式

2022-05-27 javascript Comments
  • 概述
    • JS 单线程，因为操作DOM时不能出现两个命令同时操作一个DOM的情况，JS 为浏览器而生，所以只有单线程
    • 同步与异步：
      • 单线程会造成阻塞（比如网络查询等结果），因此有两种工作模式（同步、异步），异步解决阻塞的问题
  • 函数式编程

    • 优点：可以取代带 this 的写法，react需要， 方便 treeclean （就是打包编译过滤无用代码）

    • 概念：（FP）：把过程抽象（输入与输出是映射关系）；纯函数，相同输入始终相同输出。其实就是把输入转化成唯一对应的输出

    • 函数式编程的重要概念：

      • 函数是一等公民，我觉得可以叫做把函数可以当作最小原子

        • 函数可以存在于变量中
        • 函数可以作为参数
        • 函数可以作为返回值，或者可以说用函数生成函数，其中可能会产生闭包数据，用来当作生成的函数的默认参数，也就是不用传输的参数（例子：只执行一次的函数）

      • 高阶函数，把函数作为参数**(需要练习的部分， 包括apply,bind,call这些，先往下看)**

        • forEach 把for循环用函数方式来写，写起来肯定方便，不知道有什么用
        • map
        • filter
        • every
        • some
        • find/findIndex
        • reduce
        • sort

      • 闭包 （两个案例，其实两个案例差不多，就是要灵活掌握用函数生成函数，同时添加进默认参数，需要练习）

        • 内部作用域中的函数可以访问外部作用域中的函数中的元素，这个元素不会被释放内存得以保留（例子就是之前的once函数）
        • 应用：用函数生成自带默认参数的函数

      • 纯函数，不能把纯函数看作映射关系，应该是不太好的说法，容易误解

        • wiki: 相同参数的函数返回值相同；函数应用程序没有副作用（应该指的是不依赖函数外的数据）。

        • slice 与 splice

          • slice是纯函数，splice不是纯函数。

        • lodash库，都是纯函数

          • first

          • last

          • toUpper

          • reverse

          • each

          • includes

          • find

          • fundIndex

        • 纯函数的好处

          • 可缓存，用于提高程序性能，原理是执行函数后把数据封存起来（好像也就是个闭包把数据封存了，好像没啥东西）（写个memory），也就是写个函数让memory封装一下成闭包
          • 可测试
          • 利于并行处理（因为不会对原数据做修改）（es6的webworker可以开启多线程，听过，没用过）

      • 副作用，如果一个函数依赖外部的状态就无法保证输出相同。我大概懂了，应该是种工程性思维，让函数互相独立，便于解耦

    • 柯里化，把一个函数固定一部分参数来生成新的函数（练习es6箭头写法）

      • 关键在于使用纯函数用函数生成函数的思想
      • 要时刻注意函数化的思想，把一切能用的函数函数化（使用curry函数来实现函数化，而不是自己写闭包）

    • 手写柯里化，本质上也是利用了闭包来保留已经输入的参数，最后合并

    • 函数组合

      • compose函数是把多个顺序执行的函数合并成一个函数，参数是函数名，从右往左依次执行

      • Lodash 中的组合函数，flowRight。手写模拟

        •  const _ = require('lodash')

 const reverse = arr => arr.reverse()
 const first = arr => arr[0]
 const toUpper = s => s.toUpperCase()

 const f = _.flowRight(toUpper, first, reverse)

      • 手写 compose 巧妙使用了reduce函数，reduce两个参数分别是value 与 函数名

      • 函数组合满足结合律，函数组合与改造必须多练习

      • 手写柯里化函数来实现函数组合，必须要多练习，非常重要

      • lodash 中 fp模块 中的函数都是柯里化函数（也许？），可用于compose （lodash 中的map 与 lodash fp 模块中的map并不一样）

    • Point Free

      • 一种编程风格，先组合函数，在直接调用一次运算
      • 感觉应该找个字符串处理的题库练习一下，或者其他业务场景的题库，不知道哪里找
      • 练习：world wild web => W. W. W

    • Functor 函子 需要练习构造函子

      • 用于管理异步操作，或者是减少副作用。应该算是工程化的合理方法
      • 总结：
        • 可以把函子理解为一个封装了值的盒子，由map方法传入一个处理值的函数，由函数处理盒子，最后返回一个新的盒子。有几次函数处理就创造几个新的盒子

    • MayBe 函子 需要练习构造函子

    • Either 函子

    • IO 函子

    • Folktal库

      • compose
      • curry，需要指明参数个数
      • Task 函子，实际应用，使用Task函子解决异步任务

    • Pointed 函子

      • 实现了of方法的静态函子（任何函子都可以加上），然后就可以避免手写new的方式来创造新的“盒子”。

    • Monad （单子）
